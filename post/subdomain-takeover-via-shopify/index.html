    • Subdomain Takeover Via Shopify
    2020-01-25 / 2 min read

    一、前言

      Shopify 是一个面向中小型企业的多渠道电商服务平台，它集建站、销售和宣传服务，帮助用户通过线上网店或社交媒体随时随地销售产品，Shopify 为全球 60 多万商家提供了线上服务，在高峰期每秒处理 8 万个请求。

    二、漏洞原理

      作为电商行业，如果在利用 Shopify 建站时，Shopify会允许指定备用域名，而备用域名时通常有两种做法：
     · HTTP 重定向301/302
     · CNAME 记录
      当企业使用 CNAME 记录的方法时，Shopify 会验证备用域名是否正确配置 CNAME 记录，但并不会验证备用域名的所有权。因此任意用户都可以将他人的备用域名作为己用进行连接，这就导致了子域名接管。

    三、利用要求

      1、企业于 Shopify 上进行建站；
      2、企业以 CNAME 的形式配置备用域名，且未进行连接。

    四、实际利用过程

     1、发现可能存在漏洞的页面；


     2、确认其为 Shopify 的某个商家的子域名；


     3、注册一个 Shopify 用户；

     4、进入后台，选择 settings ----> domains ----> Connect existing domain，尝试连接该备用域名；


     5、连接成功后即成功接管该备用域名，可以在 pages 配置相关页面。


    五、预防方法：

      采用备用域名时，添加 CNAME 记录后，需要主动登录 Shopify，连接该备用域名。
